Procedural Content with Splines

There are different things that have been done with procedural content.

Here we are going to see some basic classifications and where the different generative styles fit in. The few listed below are some important concepts that need to be understand before jumping into any particular case.

  • Generative Art

    Is the art that has been created using a random process. The artist uses the system to indicate how the result piece should look like. Consider a generative system as a tool someone uses to define rules that tells the software how to generate some content.

  • Infinite Games

    Many games or graphics projects create the world as the player walks through new areas. Its a very interesting effect since a wide range of algorithms are available. A very intersted concept is the Level of Detail (LOD), since the world need to be optimezed and just render in the case of the player is using it.

  • Random Generation

    Within every generative process random numbers are involved, but these are usually not enough. A seed is requierd to add a more natural look to the thing we are trying to generate. Perlin noise works great if used with different seeds to achive unique textures or functions that can later be applied to meshes for example.

  • Procedural generation

    It is about letting algorithms create content, and in some specific cases, in real-time. In the case of LOD, highly optimized systems are required. We do not want to render or generate any of the things the player is net able to see.

Keep reading the following blog posts for more about lines and the things you can do with them. Some of the use cases where the procedural generation has been useful:

  • shapes
  • endless generation
  • motion graphics
  • hills
  • nodes
